# Internationalization convention

Locales are registered in `src/i18n/locales.ts`. Message files are split by product domain so translators and developers do not edit one oversized JSON file.

```text
messages/
  en/
    common.json
    home.json
    marketplace.json
    skill-detail.json
    leaderboard.json
    dashboard.json
    auth.json
    seo.json
  vi/
    ...same filenames and keys...
```

Rules:

- English and Vietnamese must have identical namespaces and keys.
- UI copy belongs to its feature file; shared navigation and footer copy belongs to `common.json`.
- Titles, descriptions, breadcrumb labels, and schema labels belong to `seo.json`.
- Components never branch on `locale === "vi"` to choose sentences. Use `getTranslations` or `useTranslations`.
- Database content is not translated by message files. When localized catalog content is added, model it explicitly in Supabase rather than hiding translations in components.

## Adding Chinese later

1. Add `zh` to `localeConfig` in `src/i18n/locales.ts`.
2. Add `loadChineseMessages()` and a `zh` case in `src/i18n/messages.ts`.
3. Copy `messages/en/` to `messages/zh/`, translate values, and keep every key unchanged.
4. Update any localized Supabase content strategy before translating catalog records.
5. Verify `/zh`, hreflang output, sitemap entries, Open Graph locale, language switching, and fallback behavior.
